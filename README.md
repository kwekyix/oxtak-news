# Oxtak Press & Coverage Blog

The press page at **oxtak.com/blog** — a curated list of media mentions of Oxtak, shown as a filterable, searchable timeline.

---

## One-time setup

You only need to do this once.

**1. Install Python**
Download and install Python 3.10 or newer from [python.org](https://www.python.org/downloads/). During installation, check the box that says **"Add Python to PATH"**.

**2. Install dependencies**
Open a terminal in this folder and run:
```
pip install -r requirements.txt
```

**3. Get a SerpAPI key (free)**
- Go to [serpapi.com](https://serpapi.com/users/sign_up) and sign up (no credit card needed)
- After signing in, copy your API key from the dashboard
- In this folder, duplicate `.env.example`, rename the copy to `.env`, and replace the placeholder with your key:
```
SERPAPI_KEY=paste_your_key_here
```
The free plan gives 100 searches/month, which is more than enough.

> `.env` is your personal key file — never share it or commit it to git. `.env.example` is safe to share and is already in the repo.

## How to update the blog

Run these four commands in order from this folder:

```
Step 1:  python oxtak_scraper.py           -> searches the web for new mentions
Step 2:  python build_blog.py --review     -> you decide what to approve or reject
Step 3:  python build_blog.py              -> builds the blog files
Step 4:  upload the public/ folder to oxtak.com/blog
```

### What the keys do in build_blog.py --review
| Key | What happens |
|-----|--------|
| `Y` | Approved — will appear on the blog |
| `N` | Rejected — discarded permanently |
| `S` | Skip — will appear again next time |

## Adding a mention manually

If you find an article yourself and want to add it directly without running the scraper, edit `oxtak_approved.json` and add an entry at the end of the list (before the closing `]`):

```json
{
  "url":         "https://example.com/article",
  "source":      "Example Media",
  "source_type": "tech_media",
  "title":       "Article title",
  "date":        "2026-06-15",
  "snippet":     "Short excerpt that will appear on the card.",
  "language":    "en"
}
```

Don't forget the comma after the previous entry. Then run:

```
python build_blog.py
```

**Valid `source_type` values:**

| Value | Use for |
|-------|---------|
| `tech_media` | Tech publications (NotebookCheck, The Verge, TechCrunch…) |
| `news` | General news sites |
| `blog` | Blog posts |
| `social_x` | X / Twitter posts |
| `social_linkedin` | LinkedIn posts |
| `social_facebook` | Facebook posts |
| `social_reddit` | Reddit posts |
| `video_youtube` | YouTube videos |

## Removing a mention permanently

1. Open `oxtak_approved.json` and delete that entry
2. Open `oxtak_scraper.py`, find `BLOCKED_URLS`, and add the URL there so it never comes back
3. Run `python build_blog.py`

## Files — what to edit and what not to

| File | Can you edit it? |
|------|---------------|
| `oxtak_approved.json` | Yes — this is the main list you curate |
| `oxtak_mentions.json` | Yes — raw scraper output, safe to edit |
| `public/index.html` | Yes — page layout |
| `public/style.css` | Yes — visual design |
| `public/app.js` | Yes — page behaviour |
| `public/oxtak_data.js` | No — auto-generated by `build_blog.py`, changes will be overwritten |
| `.env` | Yes — your API key lives here, never share this file |
| `.env.example` | Yes — update this if the required keys ever change |

## How the scraper works

When you run `python oxtak_scraper.py`, it does the following:

1. **Search** — Runs 5 search terms through SerpAPI's Google News engine (`"Oxtak"`, `"Moneypenny oxtak"`, `"Oxtak AI recorder"`, `"Oxtak Moneypenny"`, `"Oxtak recorder review"`). SerpAPI returns real article URLs directly.

2. **Filter** — Keeps only results where the title or snippet contains `"oxtak"` or `"moneypenny"`. Discards anything unrelated.

3. **Deduplicate** — Removes duplicate URLs that appeared across multiple search terms.

4. **Skip already-approved** — Reads `oxtak_approved.json` and removes any URL already in there. This means it's safe to run anytime — it will only ever surface genuinely new articles.

5. **Save** — Appends new finds to `oxtak_mentions.json` without touching entries that are already there. Any manual edits you've made to that file are preserved.

After this, run `python build_blog.py --review` to go through the new finds one by one and approve or reject them.

**To block a URL from ever appearing again**, add it to `BLOCKED_URLS` at the top of `oxtak_scraper.py`.

## Folder structure

```
oxtblog/
├── oxtak_scraper.py       ← searches the web, writes oxtak_mentions.json
├── build_blog.py          ← review + build pipeline
├── oxtak_approved.json    ← the curated list (edit this freely)
├── oxtak_mentions.json    ← raw scraper output
├── requirements.txt       ← Python dependencies
├── .env.example           ← template showing what goes in .env
├── .env                   ← your API key (never commit this, not in repo)
└── public/                ← upload this entire folder to oxtak.com/blog
    ├── index.html
    ├── style.css
    ├── app.js
    ├── oxtak_logo.svg
    └── oxtak_data.js      ← auto-generated, do not edit manually
```
